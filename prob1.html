  1. What is $F_8$?

    We can simply list out the numbers: $1, 1, 2, 3, 5, 8, 13, 21$.

    Therefore $F_8 = \boxed{21}$.

  2. Write a function that takes a single positive integer $n$ and returns $F_n$.
    This piece of code pretty much is exactly the definition, except for the addition of fibonnaci(0) = 0 for completeness, and returning 0 for negative $n$.
    function fibonnaci(n){ if(n <= 0) return 0; else if(n <= 2) return 1; else return fibonnaci(n-1) + fibonnaci(n-2); } console.log("fibonnaci(1) = " + fibonnaci(1)); //Should be 1 console.log("fibonnaci(3) = " + fibonnaci(3)); //Should be 2 console.log("fibonnaci(8) = " + fibonnaci(8)); //Should be 21

  3. Let $N_n$ be the number of times the fibonnaci() function is called when fibonnaci(n) is invoked. $N_1 = N_2 = 1$, since the function immediately returns. $N_3 = 3$, counting the call to fibonacci(3), which in turn calls fibonnaci(2) and fibonnaci(1). Thus, $N_i = 1 + N_{i-1} + N_{i-2}$.

    The first few terms of $N_i$ are $1, 1, 3, 5, 9$. We're looking for $N_5 = 9$, so the final answer is $\boxed{9}$ invocations of the function.

    The answer could also be obtained like this:

    F(5)
 |_______________   
 |               |
F(4)            F(3)
 |_______        |________
 |       |       |        |
F(3)    F(2)    F(2)     F(1)
 |____ 
 |    |
F(2)  F(1)

    which also results in the same answer of $\boxed{9}$.

  4. As the definition of $N$ includes two previous terms of $N$, the number of calls to fibonnaci() should grow approximately as $O(2^n)$. Since exponential functions get rather big rather fast, it would be quite unreasonable to compute, say, a mere fibonnaci(100). It would take $708449696358523500000$ calls:
    var N = [0, 1, 1]; //Predefine N_1 and N_2 as 1. for(var i = 3; i <= 100; i++) //Up to N_100 N[i] = N[i-1] + N[i-2] + 1; //As defined in #3: N_i = 1 + N_(i-1) + N_(i-2) console.log("N_100 = " + N[100]);
